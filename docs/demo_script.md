# Сценарий демонстрации

## Цель
Показать потенциальному клиенту, как AI sales assistant сокращает время консультации и улучшает качество лида.

## Подготовка перед созвоном
1. Проверить `.env`: есть `TELEGRAM_BOT_TOKEN`.
2. Опционально задать `OPENAI_API_KEY`.
3. Опционально задать `MANAGER_TELEGRAM_CHAT_ID`.
4. Запустить `python -m table_sales_assistant.main`.

## Демо-сценарий 7-10 минут
1. Открыть бота и выполнить `/start`.
2. Показать главное меню: Подобрать стол, Частые вопросы, Оставить заявку, Демо-режим.
3. Пройти подбор: бюджет, рост, мониторы, use case, моторы.
4. Показать 2-3 рекомендации с объяснениями.
5. Задать FAQ-вопрос: "один или два мотора".
6. Показать ответ из knowledge базы.
7. Пройти сбор заявки до конца.
8. Открыть `LEADS_PATH` и показать сохраненный лид.
9. Если есть `MANAGER_TELEGRAM_CHAT_ID`: показать сообщение менеджеру.

## Ключевые акценты для бизнеса
- Подбор не выдумывает товары: только из структурного каталога.
- FAQ закрывает типовые вопросы без участия менеджера.
- Лид приходит с параметрами, которые сокращают время следующего звонка.
- Решение подходит для пилота и адаптации под каталог клиента.

## Что проговорить как ограничения
- Демо-данные: sample.
- Нет прямой CRM-интеграции в текущем scope.
- Реальные сроки, остатки и финальная цена подтверждаются менеджером.
