# Сценарий демонстрации

## Цель

Показать потенциальному клиенту, как AI sales assistant сокращает время консультации и улучшает качество лида.

## Подготовка перед созвоном

1. Проверить `.env`: есть `TELEGRAM_BOT_TOKEN`.
2. Опционально задать `OPENAI_API_KEY`.
3. Опционально задать `MANAGER_TELEGRAM_CHAT_ID`.
4. Запустить `python -m table_sales_assistant.main`.

## Демо-сценарий 7-10 минут

1. Открыть бота и выполнить `/start`.
2. Написать: `Привет`.
3. Написать: `рост 190 бюджет 50000 для дома`.
4. Убедиться, что бот рекомендует 1-3 модели и в каждой карточке есть:
   - почему подходит;
   - ограничения/компромисс;
   - уровень уверенности (`high/medium/low`);
   - следующий шаг (CTA).
5. Проверить, что бот рекомендует только `adjustable_desk` и не показывает аксессуары как основной стол.
6. Написать: `сравни варианты`.
7. Написать: `давай дешевле` и проверить, что список либо реально дешевле, либо есть объяснение ограничения (например, бюджет) и ближайшие альтернативы.
8. Написать: `А чем два мотора лучше?` и `Какая гарантия?`:
   - ответ должен учитывать известные параметры клиента (рост/бюджет/мониторы/сценарий/город, если есть);
   - fallback не должен быть пустым, а должен вести к следующему шагу.
9. Написать: `Оставь заявку` и пройти сбор лида до конца.
10. Открыть `LEADS_PATH` и показать сохраненный лид с контекстом консультации.
11. Если есть `MANAGER_TELEGRAM_CHAT_ID`: показать сообщение менеджеру, включая блок "Сводка для менеджера".

## Пример улучшенного ответа (рекомендация)

- `Рекомендую:`
- `1. Demo Desk ...`
- `Почему подходит: ...`
- `Ограничения: ...`
- `Уверенность: high`
- `Следующий шаг: Сравнить варианты.`

## Пример fallback без точного совпадения

- `Точного совпадения сейчас нет.`
- `Что ограничивает подбор: жесткий лимит бюджета.`
- `Ближайшие альтернативы: ...`
- `Следующий шаг: Позвать менеджера.`

## Ключевые акценты для бизнеса

- Подбор не выдумывает товары: только из структурного каталога.
- FAQ закрывает типовые вопросы без участия менеджера.
- Лид приходит с параметрами, которые сокращают время следующего звонка.
- Решение подходит для пилота и адаптации под каталог клиента.
