# Правила разработки

- Не хранить секреты в репозитории.
- Все ключи только через .env.
- Не делать vendor lock-in под одного клиента.
- Сначала simple MVP, затем расширение.
- Любая новая бизнес-логика должна быть покрыта тестами.
- Не смешивать Telegram-слой, AI-слой, каталог и хранение данных.
- Не писать монолитный код в одном файле.
- Не делать сложную архитектуру раньше времени.
- Все демо-данные должны быть явно помечены как sample.
