# Demo checklist

## Перед демо

- Установлены зависимости: `pip install -r requirements.txt`
- Пройдены проверки: `pytest`, `ruff check .`
- Настроен `.env` с `TELEGRAM_BOT_TOKEN`
- Подготовлен Telegram аккаунт для показа
- Известен путь `LEADS_PATH` для проверки сохраненного JSON

## Smoke-test сценарий

1. Отправить `/start` и проверить кнопки меню: `Подобрать стол`, `Частые вопросы`, `Оставить заявку`, `Демо-режим`.
2. Нажать `Подобрать стол` и пройти вопросы: бюджет, рост, мониторы, сценарий (опционально: можно выбрать `Пропустить`).
3. Убедиться, что бот показывает 1-3 рекомендации и кнопку `Оставить заявку по этим вариантам`.
4. Проверить в рекомендации обязательные блоки: `Почему подходит`, `Ограничения`, `Уверенность`, `Следующий шаг`.
5. Проверить fallback-сценарий без точного совпадения: бот объясняет ограничение подбора, предлагает ближайшие альтернативы и CTA менеджера.
6. Проверить FAQ: спросить по словам `мотор`, `два монитора`, `доставка`, `гарантия`, `материал`; ответ должен учитывать известные параметры пользователя при их наличии.
7. Нажать `Оставить заявку по этим вариантам` и заполнить минимум: имя, телефон, город, комментарий.
8. Проверить JSON лида в `LEADS_PATH`: есть `recommended_products` и параметры подбора (`budget`, `height_cm`, `use_case`, `monitors_count`).
9. Если задан `MANAGER_TELEGRAM_CHAT_ID`, проверить сообщение менеджеру и наличие блока: параметры клиента, рекомендованный вариант, нерешенный вопрос, следующий шаг менеджера.

## После демо

- Показать сохраненный лид в `LEADS_PATH`
- Зафиксировать обратную связь клиента
- Согласовать следующий этап пилота
