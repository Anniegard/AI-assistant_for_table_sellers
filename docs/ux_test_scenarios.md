# UX Test Scenarios

Ниже чек-лист ручных сценариев перед демо клиенту.

## 1) Классический пошаговый подбор
- Шаги:
  - "Для работы дома"
  - "178"
  - "до 50к"
  - "2 монитора"
  - "только ноутбук"
  - "140x70"
- Ожидание:
  - бот выдает рекомендацию;
  - в тексте нет технических enum-значений.

## 2) Свободная фраза с неполными параметрами
- Шаг: "рост 180 бюджет 50000 для дома"
- Ожидание:
  - бот не рекомендует сразу;
  - бот задает вопрос: "Сколько мониторов будет на столе?"

## 3) Полная свободная фраза
- Шаг: "рост 180, бюджет 50-80к, 2 монитора, системник на столе, 160x80, для игр"
- Ожидание:
  - бот может сразу выдать рекомендацию.

## 4) "Дешевле" без бюджета
- Шаг: "давай дешевле"
- Ожидание:
  - бот спрашивает: "В каком бюджете теперь смотреть? Например: до 50 000 ₽ или 40-60к."
  - мгновенного пересчета без бюджета нет.

## 5) "Дешевле" с бюджетом
- Шаг: "давай до 50к"
- Ожидание:
  - бюджет обновляется;
  - рекомендации пересчитываются без перезапуска сценария.

## 6) Смена сценария
- Шаг: "вообще это для игр"
- Ожидание:
  - сценарий обновляется;
  - подбор пересчитывается или продолжается с новым сценарием.

## 7) Ограничение по месту
- Шаг: "места максимум 120"
- Ожидание:
  - бот учитывает ограничение ширины столешницы.

## 8) Без ограничений по размеру
- Шаг: "без ограничений"
- Ожидание:
  - бот не ограничивает подбор по размеру столешницы.

## 9) Низкий бюджет
- Шаг: "5000"
- Ожидание:
  - бот уточняет, не имелся ли в виду, например, бюджет 50 000 ₽.

## 10) Заявка менеджеру
- Шаг: "оставить заявку"
- Ожидание:
  - стартует сбор контактов (имя/телефон/город);
  - уже собранные параметры подбора не теряются.
